# IP 悬停审核助手 PRD

## 1. 产品背景

在审核、风控、黑产排查场景中，审核员经常需要快速判断一个用户 IP 是否更像机房 IP、家庭/住宅网络、代理/VPN 或其他异常来源。传统做法通常是复制 IP 后跳转到第三方网站查询，效率低、流程割裂，而且在公司办公环境下往往不能依赖 VPN。

因此，需要一个安装在个人办公电脑上的轻量级 Chrome 插件：直接在公司后台页面中悬停 IP，即可弹出中文信息卡片，辅助人工审核判断。

## 2. 产品目标

- 在不依赖 VPN 的前提下，快速查询 IP 基础画像
- 提升审核员对机房/住宅/代理/VPN 的识别效率
- 尽量减少信息暴露，只发送被查询的单个 IP
- 通过中文 UI 降低使用门槛

## 3. 目标用户

- 在公司后台做人工审核、风控复核、黑产排查的个人用户
- 非技术背景但需要快速识别 IP 风险的审核人员

## 4. 核心使用流程

1. 用户在公司后台打开审核页面
2. 页面中出现用户登录 IP / 注册 IP / 操作 IP
3. 鼠标悬停到 IP 文本上
4. 插件自动查询并弹出中文卡片
5. 用户根据“网络属性 + 风险等级 + 判断依据”做进一步审核决策

## 5. 核心功能

### 5.1 IP 自动识别
- 自动扫描网页中的 IPv4 文本
- 支持表格、列表、详情页中的纯文本 IP
- 首版不要求支持 IPv6

### 5.2 悬停查询
- 鼠标停留一段时间后触发查询
- 支持“直接悬停”和“Alt + 悬停”两种方式
- 鼠标移开后自动隐藏卡片

### 5.3 中文信息卡片
展示字段包括：
- IP 地址
- 网络属性
- 风险等级
- 匿名属性
- 国家/地区
- ASN
- 运营商
- 组织名称
- 数据来源
- 缓存状态
- 判断依据

### 5.4 数据源与容灾
- 主数据源：ipapi.is
- 备用数据源：ipquery
- 支持切换主/备顺序
- 主数据源失败时自动降级到备用数据源

### 5.5 本地缓存
- 默认缓存 24 小时
- 同一 IP 重复查询时优先读本地缓存
- 支持手动清空缓存

### 5.6 域名白名单
- 允许用户只在指定公司后台域名启用插件
- 未命中白名单的网页不触发任何识别与查询逻辑

## 6. 判断输出设计

### 网络属性
- 机房/数据中心
- 家庭/住宅网络
- 代理/VPN
- 移动网络
- 企业网络
- 未知

### 风险等级
- 高
- 中
- 低

### 判断依据
示例：
- 命中机房/云服务商特征
- 命中 VPN 标签
- 命中 ISP/家庭宽带特征
- 数据源信息不足，建议复核

## 7. 非功能要求

### 安全性
- 不上传整页 HTML
- 不读取账号、手机号等无关字段
- 仅在用户触发查询时发送单个 IP 给外部数据源
- 支持域名白名单，减少误用范围

### 易用性
- 全中文 UI
- 无需独立后端服务
- 安装后可直接在本机浏览器使用

### 性能
- 查询响应尽量控制在 1~2 秒内
- 已缓存 IP 尽量做到即时展示

## 8. MVP 范围

首版必须完成：
- IPv4 自动识别
- 悬停查询
- 中文卡片展示
- 双数据源切换
- 本地缓存
- 白名单域名控制
- 设置页

首版暂不做：
- IPv6
- 批量导出
- 黑名单系统联动
- 企业后端服务
- 多人共享与审计

## 9. 验收标准

1. 能在 Chrome 中以“加载已解压扩展程序”的方式正常安装
2. 在普通测试页面或后台页面中，文本 IP 能被识别并悬停触发
3. 可以正确展示中文信息卡片
4. 当主数据源不可用时，能自动尝试备用数据源
5. 同一 IP 第二次查询能命中缓存
6. 设置页可保存白名单、触发方式、缓存时间、数据源顺序
7. 单元测试通过，数据源测试报告可生成
